# Learn Docker 🐳

A comprehensive workbook to learn Docker from the basics to advanced concepts with practical examples.

## 🎯 Learning Path

This repository is organized as a progressive learning path. Start from the beginning and work your way through each example:

### 1. Getting Started
- **01-hello-world**: Your first Docker container
- **02-basic-web-app**: Simple HTML/CSS website in Docker

### 2. Application Containers
- **03-python-app**: Python Flask application
- **04-node-app**: Node.js Express application
- **05-nginx**: Web server configuration

### 3. Advanced Concepts
- **06-multi-container**: Docker Compose with multiple services
- **07-volumes**: Data persistence and volume management
- **08-networking**: Container networking and communication
- **09-best-practices**: Optimization and security best practices

## 🚀 Prerequisites

Before starting, make sure you have:
- Docker installed on your system ([Install Docker](https://docs.docker.com/get-docker/))
- Basic command line knowledge
- Text editor of your choice

## 🛠️ How to Use This Repository

1. **Clone the repository**:
   ```bash
   git clone https://github.com/ajharris/learnDocker.git
   cd learnDocker
   ```

2. **Start with the first example**:
   ```bash
   cd examples/01-hello-world
   ```

3. **Follow the README in each directory** for step-by-step instructions

4. **Build and run each example** to see Docker in action

## 📚 Key Docker Concepts You'll Learn

- **Images vs Containers**: Understanding the difference
- **Dockerfile**: Writing instructions to build images
- **Docker Commands**: Essential CLI commands
- **Ports and Networking**: Exposing services
- **Volumes**: Persisting data
- **Docker Compose**: Multi-container applications
- **Best Practices**: Security, optimization, and maintenance

## 🎓 Learning Tips

- **Hands-on approach**: Run every example yourself
- **Experiment**: Modify the examples to see what happens
- **Read the logs**: Docker logs provide valuable debugging info
- **Clean up**: Remove unused containers and images regularly
- **Practice**: Create your own Dockerfiles for existing projects
